# サハラで袖を切れ

Podcast「限界突破ライフハック」#11(サハラ砂漠完走ランナーが服の袖を切り始めた話)を元にした、装備削ぎ落としローグライク・ブラウザゲーム。

7日間・264kmの砂漠マラソン。ザックは重量制限オーバーの9.57kgからスタートし、装備を「切る・折る・捨てる」ことで7.5kg以下に削って出走する。削った快適さのツケは、数日後のイベントで回収される。

**▶ プレイする: https://hirokiinoue0602.github.io/sahara-sode-kire/**

## 遊び方(ローカル起動)

ビルド不要の静的ファイル構成。任意の静的サーバーで配信するだけで動く。

```sh
python3 -m http.server 8931 --directory game
# → http://localhost:8931
```

デプロイも `game/` の中身をそのまま静的ホスティング(GitHub Pages / Cloudflare Pages / S3 等)に置くだけ。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` | エントリポイント |
| `data.js` | ゲームデータ: 装備(GEAR)・日程(DAYS)・イベント(DAY_EVENTS / NIGHT_EVENTS)・就寝回復(SLEEP_RECOVERY)・実績(ACHIEVEMENTS) |
| `main.js` | エンジン+UI: 画面遷移(title → prep → run → result)、シード付き乱数、効果適用、localStorage保存 |
| `style.css` | スタイル。bodyクラス(phase-title / prep / day / night)で昼夜の見た目が切り替わる |

## ゲームデザインの要点

- **リスクはタグで伝搬する**: 装備の削り段階が `tags`(sunburn, cold, medkit, foodlv…)を持ち、イベント側が `h.tag(s, name)` で参照して結果を分岐する。装備とイベントは疎結合なので、どちらも独立に追加できる。
- **レース中にも切れる**: HUDの「✂装備」から走行中に装備を切れる(戻せない)。タイトル回収要素。
- **デイリーシード**: 「今日のサハラ」は日付文字列のシードで全員同じイベント列になる。

## バランス調整の勘所

自動プレイ検証時の完走率の目安(2026-07-14時点):

- 食料+ストーブだけ削る手堅いビルド → 完走率 ほぼ100%(体力残 55〜67)
- 中間ビルド → 完走率 ほぼ100%(体力残 36〜54)
- 寝袋を全部切るビルド → 完走率 約50%
- 全装備を限界まで切る → 完走率 0%(184〜243km地点でリタイア)

主な調整ノブ:

- 日次ドレイン: `main.js` の `stepDayStart()`(`staDrain = 10 + kg * 1.9` など)
- 就寝回復: `data.js` の `SLEEP_RECOVERY` と `stepSleep()` の寒さペナルティ(`cold * 3`)
- 個別イベントのダメージ: `data.js` の各イベント
